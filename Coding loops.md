So I am setting up coding loops for agents to follow.

The process is:
1. Defining a spec by discussing with an agent using autonomous coding repo
2. Once spec is defined, converting it into a list of features, which are to be implemented, with name, completion state, testing procedure
3. Setting up a loop with a common prompt to execute  a rotating loop - implementation, testing for 2-3 features at a time
4. The loops are in tmux, so you can tmux attach and steer the loops

## Build-Verify Loop

The current approach uses a **two-agent pipeline** per cycle: a builder and an independent verifier. This replaces the old single-agent loop.

### How it works

Each cycle the shell script picks the next N tickets (default: 2), then:

1. **Builder** — reads the assigned tickets, implements them, runs the verification commands from each ticket, and appends a result block to `build_report.md`. Outputs `BUILD DONE: COMPLETE / PARTIAL / FAILED`.
2. **Verifier** — independently reruns the same verification commands, tests corner cases beyond what the ticket specifies, and reads the code for sanity. If all pass: commits. If any fail: reports detailed failure notes for the builder to read on the next retry. Outputs `VERIFY DONE: PASS / FAIL`.

The script tracks ticket state in `.ticket_progress` (COMPLETE / IN_PROGRESS / FAILED:N / INTERRUPTED). Failed tickets are retried up to `--max-retries` times before being skipped.

### Inputs required

| File | Generated by |
|---|---|
| `prompts/app_spec.txt` | `/code-create-spec` skill |
| `prompts/tickets.md` | `/code-create-spec` skill (final step) |
| `prompts/builder.md` | `/code-build-loop` skill |
| `prompts/verifier.md` | `/code-build-loop` skill |
| `scripts/run_build_verify_loop.sh` | `/code-build-loop` skill |

### Starting the loop

```bash
bash scripts/run_build_verify_loop.sh \
  --builder-prompt prompts/builder.md \
  --verifier-prompt prompts/verifier.md \
  --tickets prompts/tickets.md \
  --run-name <name> \
  --max-retries 3 \
  --tickets-per-session 2
```

Monitor: `tmux attach -t agent-<name>`
Stop cleanly: `touch scripts/agent-run-logs/<name>/.done`

### Logs

All output goes to `scripts/agent-run-logs/<run-name>/`:
- `loop.log` — shell-level cycle log
- `build_report.md` — running record of every builder and verifier result
- `.ticket_progress` — current state of each ticket
- `builder_cycle_N.log` / `verifier_cycle_N.log` — full agent output per cycle
