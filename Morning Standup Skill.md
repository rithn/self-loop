# Morning Standup Skill (`/plan-standup`)

> Type `/plan-standup` to run your daily standup — reviews yesterday, updates context files, plans today with a time-blocked schedule, syncs calendar, and updates HubSpot.

---

## What it does

An automated daily standup that pulls context from every relevant source before asking a single question. It reads git logs, email, calendar, meeting transcripts, and HubSpot to auto-resolve as much as possible before surfacing batch questions. Produces a time-blocked task file for the day.

**If today's task file already exists**, the skill shows it and stops — it won't overwrite work already done.

---

## Data sources fetched on startup

All of the following are pulled in parallel before Step 1 begins:

| Source | What it provides |
|---|---|
| `Context System/projects.md` | Project priorities, statuses, deadlines, loop candidates |
| `Context System/company.md` | Company context |
| `Daily tasks/[yesterday].md` | Yesterday's task list (fallback: most recent available) |
| `Clients & Partners/` (subagent) | Pending contracts, commitments, awaiting-response items — distilled summary |
| `git log` per active project | Actual commit history since yesterday — primary source for build status |
| Google Calendar (yesterday + 14 days) | Events, meetings, conflicts |
| Calendar conflict subagent | Flags: stale project statuses, day-of-week mismatches, uncaptured meeting outcomes |
| Gmail digest (`email_raw_[today].json`) | Raw Gmail threads with full bodies — generated by `fetch_email_digest.py` before standup |
| Fireflies transcript (`fireflies_[yesterday].md`) | Raw meeting transcript — generated by `fetch_fireflies_transcripts.py` |
| HubSpot open tasks (subagent) | All open tasks for each team member, overdue flagged |

---

## Step 1 — Yesterday review

Prints a compact summary of yesterday's tasks and meetings. Before asking anything, runs four automatic checks:

- **Gmail digest check**: Reads the raw email JSON and generates the digest inline. Checks all same-day messages on each thread — not just the most recent.
- **Gmail inbox check**: For carry-forwards marked "awaiting response", scans the digest for replies and reports inline.
- **Calendar conflict check**: Reports flagged items from the calendar conflict subagent.
- **OOO blocker check**: If an OOO reply is found for a client with a pending deliverable, flags it as a delivery blocker.

**Auto-resolution rules (applied before asking any question):**
- Build/code tasks → checks git log. Commits found → marks as built, no question asked.
- Send/share tasks → checks sent folder in digest. Sent email found → marks as sent, no question.
- Passive carry-forwards ("awaiting X", "pending X response") → never escalated as action items unless the awaited reply arrived.
- Stale awaiting-response items (7+ days old) → surfaced automatically as overdue action items.

Only items that can't be auto-resolved are asked about, in batches of 3–4. After the final batch, always asks: "Anything completed yesterday that wasn't on the list?"

Other team members' open HubSpot tasks are printed here for visibility — not actioned on.

---

## Step 2 — Context updates

Updates `Context System/projects.md` from each project's actual state (`CLAUDE.md`, `.ticket_progress`, `build_report.md`). Updates relevant client files (`Clients & Partners/[Client].md`) with meeting outcomes and next steps. If a Fireflies transcript exists, uses it as the primary source for decisions and commitments. Creates new client files for contacts not yet in the system.

---

## Step 3 — Today's plan

Generates a time-blocked daily schedule based on:
- Carry-forwards from yesterday
- Current project priorities in `projects.md`
- Calendar blocks confirmed for today
- Soumya's open HubSpot tasks (overdue at the top, due-today as manual items)

**Planning rules:**
- Starts focused work at ~10:00 AM — standing 2h personal buffer (breakfast, chores, lunch) is always scheduled first
- If there's a major call today, its prep is folded into the meeting task and scheduled first
- Loop candidates are proactively identified from every active project — never says "none" without checking
- Items that don't fit in 8h are deferred to carry-forwards, not squeezed in

Asks for confirmation, then writes `Daily tasks/[today].md` in time-blocked format:

```
# [Date] — Daily Tasks

## Schedule
- HH:MM–HH:MM  [Project] Task description
- HH:MM–HH:MM  Breakfast / Lunch / Chores
...

## Meetings
- [Time] — [Meeting name / person]

## Background Loops
- [Project] — [what the loop is doing]

## Carry-forwards
- [Items deferred to tomorrow]
```

---

## Step 4 — Calendar check

Cross-references the plan against the 14-day calendar. Proposes specific changes (which events to move, to when, and why). Only modifies events where the calendar creator email matches the configured account — never touches external invites.

---

## Step 5 — HubSpot sync

Reviews the standup conversation for anything warranting a HubSpot update. Proposes deal stage changes, new follow-up tasks, and new contacts for approval before pushing anything.

**Rules:**
- Checks for duplicate tasks before proposing new ones
- Never proposes tasks due today (already on the task list)
- Searches by company name before updating or creating
- Only advances a deal to "Workshop agreed" if a workshop was explicitly scheduled — an intro call doesn't qualify
- Never logs meeting notes (Fireflies → HubSpot native connector handles transcripts)

---

## Step 6 — Standup retrospective

Reviews the standup conversation for gaps in the skill itself — missed data, extra back-and-forth that better instructions would have prevented, incorrect assumptions. Proposes improvements and, with confirmation, updates `~/.claude/commands/plan-standup.md` directly.

---

## Late standup

If it's past 18:00 IST, the skill notes "Late standup — today is effectively done" at the top and adjusts Step 3 to plan for tomorrow rather than today.

---

## Weekend handling

On Saturday or Sunday, the skill asks upfront: "Is today a full workday or a lighter/strategic day?" and adjusts the plan accordingly — no full client-facing task list is assumed.
